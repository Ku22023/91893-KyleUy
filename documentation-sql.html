SQL Examples and Explained

What is SQL?

SQL (Structured Query Language) is the standard language for interacting with databases. You use it to ask questions (queries) and make changes to the information stored within.

SELECT (Retrieve Data):

The SELECT command is your primary tool for getting information out of a database table.

--Get all information from the 'Students' table

SELECT * FROM Students;


--Get only the 'Name' and 'Major' columns from the 'Students' table

SELECT Name, Major FROM Students;

CREATE TABLE (Make Tables):

The CREATE TABLE command is used to define a new table in your database, specifying the names and types of the columns it will contain.

--Create a table named 'Subjects'

CREATE TABLE Subjects (

CourseID INT PRIMARY KEY,

CourseName VARCHAR(255) NOT NULL,

Credits INT

);

INSERT INTO (Add Data):

The INSERT INTO command lets you add new rows of data into an existing table.

--Add a new student to the 'Students' table

INSERT INTO Students (StudentID, Name, Subject)

VALUES (22186, 'Jacky Mo', 'DTP');


--Add a new course to the 'Subjects' table, specifying only some columns

INSERT INTO Courses (SubjectName, Credits)

VALUES ('12DTP', 22);

WHERE (Filter Data):

The WHERE clause is used to filter records, allowing you to retrieve only the data that meets specific conditions.

--Select all students who are in level 2

SELECT * FROM Students WHERE level = '2';


--Select subjects with more than 3 credits

SELECT SubjectName FROM Courses WHERE Credits > 3;

UPDATE (Change Data):

The UPDATE command allows you to modify existing records in a table.

--Update the level of the student named 'Jacky Mo'

UPDATE Students SET LEVEL = '3' WHERE Name = 'Jacky Mo';


--Update the credits of a course with a specific ID

UPDATE Courses SET Credits = 4 WHERE CourseID = 205;

DELETE FROM (Remove Data):

The DELETE FROM command is used to remove existing records from a table based on a specified condition.

--Delete the student with the ID 22023

DELETE FROM Students WHERE StudentID = 22023;


--Delete all records from the 'Enrollments' table (use with caution!)

DELETE FROM Enrollments;

ORDER BY (Sort Results):

The ORDER BY clause is used to sort the result-set of a `SELECT` statement. You can sort the records in ascending (`ASC`) or descending (`DESC`) order.

--Select all students and sort them by Name alphabetically

SELECT * FROM Students ORDER BY Name;


--Select all courses and sort them by Credits in descending order

SELECT * FROM Courses ORDER BY Credits DESC;

Joins (Combine Tables):

JOIN clauses are used to combine rows from two or more tables based on a related column between them. Common types include:

--INNER JOIN: Returns records that have matching values in both 'Students' and 'Enrollments' tables

SELECT Students.Name, Enrollments.SubjectID

FROM Students

INNER JOIN Enrollments ON Students.StudentID = Enrollments.StudentID;



--LEFT JOIN: Returns all students and any courses they are enrolled in (if any)

SELECT Students.Name, Enrollments.CourseID

FROM Students

LEFT JOIN Enrollments ON Students.StudentID = Enrollments.StudentID;

Keys:

SQL keys are constraints that ensure data integrity and establish relationships between tables. Common types include:

PRIMARY KEY

--Uniquely identifies each record in a table (e.g., StudentID in Students, CourseID in Courses).



FOREIGN KEY

--A field in one table that refers to the PRIMARY KEY in another (e.g., StudentID in an 'Enrollments' table referencing StudentID in 'Students').



UNIQUE

--Ensures that all values in a column are different (e.g., a unique student email address).



NOT NULL

--Ensures that a column cannot have a NULL value (e.g., a student's name must be provided).
