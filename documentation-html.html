HTML Examples and Explained

What is HTML?

HTML is the basic language for creating web pages. It uses simple tags to structure text, images, and links, telling web browsers how to display content. It stands for Hyper-text-markup-language

Basic Structure:

Basic Structure: Think of this as the skeleton of your webpage. You've got the <html> tag as the root, then the <head> for behind-the-scenes info like the title, and the <body> where all the visible content goes.

<html>

<head>

#behind-the-scenes items

</head>


<body>

#What you would see on the page

</body>

</html>

Headings:

Headings: These are like titles and subtitles for your content. You use <h1> for the main heading, <h2> for the next level down, and so on, all the way to <h6>.

<h1>

I am the Main Header

</h1>


<h2>

I would be slightly smaller

</h2>


<h3>

I would be smaller

</h3>


<h4>

I would be smaller

</h4>


<h5>

I would be smaller

</h5>


<h6>

I would be smaller

</h6>

Paragraph Tags:

Paragraphs: For regular blocks of text, you wrap them in <p> tags.

<p>

I am a paragraph

<p>

Links:

Links: Want to connect to other pages or websites? The <a> tag is your friend. You use the href attribute to specify the destination.

<a href="google.com">

Click me to open a link!

<a>

#Replace google.com with the URL of what you would like to link to.

Images:

Images: To show pictures, you use the <img> tag. The src attribute tells the browser where to find the image file.

<img src="image.png">

#Replace image.png with the filename of the image.

Lists:

Lists: Need to present items in a structured way? You've got two main types:

  • Unordered lists (<ul>) with bullet points (<li> for each item).
  • Ordered lists (<ol>) with numbers (<li> for each item).

#Unordered List

<ul>


<li>

Red

</li>


<li>

Orange

</li>


<li>

Yellow

</li>

</ul>

#Ordered List

<ol>


<li>

Apple

</li>


<li>

Orange

</li>


<li>

Banana

</li>

</ol>

Divs:

Divs: Think of a <div> as a container or a section on your webpage. It's like drawing an invisible box around a group of things (text, images, other elements) to organize them. <div> elements are block-level, which means they automatically take up the full width available and start on a new line.

<div>

#content inside of the Div.

</div>

Semantic Tags:

Semantic Elements: These tags give more meaning to the structure of your content, making it clearer for both browsers and developers.

<nav>

#Includes a list of links where you can navigate the website.

</nav>



<header>

#Includes the nav bar and the main title of the page.

</nav>



<main>

#Includes the majority of the body

</nav>



<footer>

#Found at the bottom of the page, giving details such as copyright info, contact info, etc.

</nav>



<article>

#Used to split Sections into further parts.

</nav>



<section>

#Used to split up the main into parts.

</nav>



<aside>

#Used to show content which is not part of the main content.

</nav>

Metadata:

Metadata: This is information about your HTML document that isn't displayed on the page itself. It lives within the <head> section. Key examples include:

<title>

#The title that appears in the browser

</title>



<meta>

#Can define things such as: Description, Keywords, and Charset of a page.

</meta>
