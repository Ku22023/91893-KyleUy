Level 1 Level 2 Level 3

Level 1 Programme

Total Credits: 20

Standards:

  • 92004: Create a Computer Programme (5 Credits, Internal)
    • Purpose: Students are able to create a computer programme.
    Click to Expand Requirements for Achieved:
  • Using a programming language to create a programme which performs a specific task.
  • The Student has tested and debugged the programme to make sure it works on expected cases.
  • The Student has provided proper documentation and has added comments to their code.
    • Requirements for Merit:
  • The Student has used proper and descriptive names for variables.
  • The Student has added comments which explain the purpose of code sections.
  • The Student has tested and debugged the programme to make sure it works in expected and boundary cases.
    • Requirements for Excellence:
  • The Student has used conditional statements and control structures effectively.
  • The Student has used constants, variables, or derived values in place of literals to make the programme flexible.
  • The Student has Tested and Debugged the programme to make sure it works, in, expected, boundary, and invalid cases.
    • The computer programme must:
  • Have at least two variables which store values
  • Take input from the user, sensor, or somewhere else
  • Produce an Output
  • Use conditionals in the code eg (if, else) and other control structures
  • Use data which was previously taken in and stored

  • 92005: Develop a digital technologies outcome (5 Credits, Internal)
    • Purpose: Students are able to develop a digital technologies outcome.
    Click to Expand Requirements for Achieved:
  • Describing the purpose, potential user, requirements, and specifications of the outcome.
  • Use appropriate tools or techniques of a digital technologies domain to produce an outcome that addresses the requirements and specifications.
  • Testing the outcome to ensure basic functionality.
    • Requirements for Merit:
  • Following conventions relevant to the tools or techniques of a digital technologies domain.
  • Using information from testing to make improvements to the outcome's fitness for purpose.
    • Requirements for Excellence:
  • Following conventions relevant to the tools or techniques of a digital technologies domain.
  • Using information from testing to make improvements to the outcome's fitness for purpose.
  • 92006: Demonstrate understanding of usability in human-computer interfaces (5 Credits, External)
    • Purpose: Students are able to develop a digital technologies outcome.
    Click to Expand Requirements for Achieved:
  • Describe the purpose of human-computer interfaces.
  • Describe how usability princiles are used in human-computer interfaces.
    • Requirements for Merit:
  • Explain how usability principles have been used in human-computer interfaces.
  • Explain the usability of human-computer interfaces using usability principles.
    • Requirements for Excellence:
  • Compare the usability of human-computer interfaces.
  • Apply usability principles to suggest improvements to human-computer interfaces.
  • 92007: Design a digital technologies outcome (5 Credits, External)
    • Purpose:Students are able to design a digital technologies outcome.
    Click to Expand Requirements for Achieved:
  • Describe a need or opportunity, potential end-user's, and requirements of the design.
  • Make design ideas for the proposed outcome.
  • Describe how the finished design addresses the need or oppotunity and meets the requirements set.
    • Requirements for Merit:
  • Use feedback to make improvements throughout designing.
  • Explain how design decisions made improved the quality of the outcome.
    • Requirements for Excellence:
  • Justify how the descisions made during the process helped the suitablility of the outcome.
